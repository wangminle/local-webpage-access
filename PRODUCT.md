# Product

## Register

product

## Users

开发者与小型主机运维者：在局域网内的 4G/8G 内存小主机（迷你 PC、NUC、树莓派级设备）上，把打包好的网页项目一条命令托管起来。他们通常在另一台笔记本上打开管理页，边部署边排障——典型上下文是「刚 import 了一个 zip，想立刻看到它跑没跑起来、端口是多少、日志有没有报错」。技术熟练，排斥花哨界面，要的是密度与准确。

## Product Purpose

`lwa` 是一个本地网页部署基座：zip 导入 → 自动识别运行形态（纯静态 / 前端 SPA / 含后端的全栈）→ 分配端口并对局域网暴露访问入口；静态走共享网关（Caddy 优先，内置 `http.server` 兜底），全栈走自动生成的 Docker Compose。管理页是这套基座的「控制台」：一眼看清哪些实例在跑、哪些出错、端口池与整机资源水位，并能对单个实例做启动 / 停止 / 重启 / 重建与日志查看。成功标准：打开页面 3 秒内知道集群健康度，10 秒内能定位任何一个出错实例并展开处理。

## Brand Personality

克制、准确、可信。三个词：**quiet**（不喧哗）、**precise**（精确）、**trustworthy**（像基础设施一样可靠）。语气是运维仪表盘，不是营销首页——数据本身说话，装饰让位给可读性。参考 GitHub Primer、Linear、Caddy / Docker 官方控制台的「工具感」：熟悉到不需要学习，密度高但不拥挤，状态色有节制。

## Anti-references

- **SaaS 营销首页的观感**：大留白、渐变、emoji 图标、「hero 大数字」模板。这是控制台，不是落地页。
- **彩色侧边条**（`border-left` 状态色）作为卡片或行的装饰性强调——典型的 AI 默认动作，明确杜绝。
- **玻璃拟态 / 模糊背景**作为默认观感。
- **渐变文字、全大写带字距的小标题（eyebrow）**堆在每个区块上方。
- **过度的进场动画 / 编排式加载序列**：用户在任务中，不想看页面「表演」加载。
- **占位用 emoji 当图标**。

## Design Principles

1. **密度优先，留白服务于扫读**：每屏传递更多有效信息，但不靠拥挤——靠分层与权重。
2. **状态色是信号不是装饰**：颜色只用于语义（运行 / 失败 / 待处理 / 进行中），绝不用于点缀；且状态不能只靠颜色传达。
3. **熟悉即正确**：采用工具型 UI 的成熟范式（顶部栏 + 表格 + 抽屉），不发明新交互；用户不该「学习」界面。
4. **每个交互态完整**：default / hover / focus / active / disabled / loading / error 都要有，键盘可达。
5. **动效传达状态，不传达个性**：只在反馈与状态变化时用 100–200 ms 短过渡，不做进场编排。

## Accessibility & Inclusion

- 目标 **WCAG 2.1 AA**：正文 ≥ 4.5:1，大字 / 图标 ≥ 3:1；muted 文本不得低于 4.5:1（「为了优雅用浅灰」是最常见的可读性失败）。
- **全键盘可达**：所有操作键盘可触达，可见 focus 环，Esc 关闭抽屉与模态（已实现）。
- `prefers-reduced-motion`：所有过渡提供降级（瞬时或极短）。
- `prefers-color-scheme`：同时提供明 / 暗主题。
- **色盲友好**：状态不仅靠颜色——badge 同时带文字与状态点，行级状态有背景与徽章双重提示。
