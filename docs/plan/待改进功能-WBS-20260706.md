## IMP-006 路径别名路由（显式启用）

| 子任务 | 触点 | 说明 |
| --- | --- | --- |
| 006.01 | `cli.py`、`importer.py` | `lwa import ... --path-alias <slug>` 可选；未传时**默认行为不变**（仅 hostPort 访问）。alias 写入 manifest `network.routeMode=port|name` + `network.routeHost`。 |
| 006.02 | `paths.py`/`security.py` | 新增 `validate_path_alias(slug)`：`^[a-z0-9]+(-[a-z0-9]+)*$`；全局唯一（registry 扫描）；保留字（`api`、`static-gateway`、`inbox`、`apps`、`registry`、`run`、`manager`、`logs`、`skills`、`templates`、`health`）拒绝。 |
| 006.03 | `static_gateway.py` | Caddy 模式主配置注入 `/<alias>/* → reverse_proxy localhost:<port>` 的 strip-prefix 路由；`/<alias>` 301 到 `/<alias>/`。builtin 模式不支持 alias（CLI 显式提示，仍只给端口）。 |
| 006.04 | `lifecycle.py` | `remove_instance` 清理别名路由；`restart_instance`/`stop_instance` 保留 alias 登记、按需禁用 upstream（V1：stop 即整站 disable，alias 随之不可达，简化处理）。 |
| 006.05 | `cli.py` | `lwa start` 输出端口 URL 与（有别名时）路径 URL。 |
| 006.06 | `tests/` | 不传别名默认行为不变；传别名端口 + 路径并存；冲突/保留字拒绝；remove 后路由清理；Caddy 生成文本断言。 |

**SPA 限制**：写死 `/assets/...` 绝对路径的 SPA 在子路径下资源 404；V1 在 `--path-alias` 帮助文本中明确提示，不强行改写 base。文档同步 `known-limitations.md`。

| 008.01 | `updater.py`（新建） | 识别 workspace（`local-web.yml`）与 repo（editable 安装路径 / 当前 git 根 / `--repo`）；预检：版本、manager/daemon 运行态、running 实例、待同步 skills/templates 清单。 |

## IMP-009 实例 zip 包更新（再导入识别与原地升级）

| 子任务 | 触点 | 说明 |
| --- | --- | --- |
| 009.01 | `importer.py` | 新增 `Importer.update_zip(zip_path, instance_id, *, restart=True, keep_data=True, yes=False, dry_run=False)`：校验实例存在 → 计算 hash 与 `sourceZipHash` 比（相同跳过 + 提示）→ sanitize（复用 001）→ 备份原 zip `source/original.zip.bak` → 清空 `current/`（保留 `data/`）→ 解压新内容 → 重扫 → **kind/runtime 变化拒绝**（首版，除非 `--force-kind-change`）→ 更新 manifest（保留 id/hostPort/data/desiredState，刷 `sourceZipHash`/`updatedAt`）→ registry 同步。失败回滚 `current/` + manifest。 |
| 009.02 | `cli.py` | `lwa import <zip> --update <id>`；slug 冲突不再 silent 建 `-2`：提示「已存在，加 `--update` 或换 `--name`」。`--yes` 非交互；`--dry-run` 展示 hash diff。 |
| 009.03 | `lifecycle.py` | 更新成功后若 `desiredState=running`：`stop`（或仅停网关）→ `rebuild`/`host_instance` → 恢复 running，**复用既有 hostPort**（不重新分配）。事件链：`update` 事件带旧 hash→新 hash。 |
| 009.04 | `manager_api.py`、`manager_static/` | `POST /api/instances/{id}/update`（body `{ "zipPath": "inbox/foo.zip" }`），复用 CLI 同一套 Python 逻辑；列表/详情可展示 `sourceZipHash` 前 8 位。 |
| 009.05 | `skills/lwa-import-zip/` | 文档化「同项目新版本 → `--update <id>`」；助手收到新 zip 先查 registry 是否同 slug，询问或带 `--update`。 |
| 009.06 | `tests/test_importer.py` | hash 相同跳过；kind 变化拒绝；失败回滚保留 `current/` 与 hostPort；`data/` 持久文件更新后仍在；不传 `--update` 且 slug 冲突报错而非建 `-2`。 |

**与 008 区分**：008 = 工具升级；009 = 业务 zip 升级。可组合：`lwa update` 后 `lwa import --update`。
